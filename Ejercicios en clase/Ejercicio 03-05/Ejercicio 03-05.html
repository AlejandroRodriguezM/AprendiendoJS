El replace()método de la Location interfaz reemplaza el recurso actual con el de la URL proporcionada. La diferencia con el assign()método es que después de usar replace()la página actual no se guardará en la sesión History, lo que significa que el usuario no podrá usar el botón Atrás para navegar hasta ella.
